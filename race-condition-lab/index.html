⚠️ About This Lab

This lab demonstrates TOCTOU (Time-of-Check to Time-of-Use) vulnerabilities. Each endpoint intentionally introduces a 2-second delay between checking a condition and updating state. This simulates race conditions where multiple concurrent requests can exploit the window between decision and action.

Vulnerability Examples:

  • Transfer: Multiple threads can transfer more money than available balance
  • Coupon: Multiple threads can use the same single-use coupon
  • Checkout: Stock can go negative by selling more than available

Use concurrent request tools (Python, Node.js, or curl scripts) to exploit these vulnerabilities!
